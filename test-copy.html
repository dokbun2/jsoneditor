JSON 복사 기능 테스트

테스트 방법

  1. http://localhost:5173/ 에서 JSON 에디터를 엽니다
  2. 아래 테스트 케이스의 JSON을 입력 필드에 붙여넣습니다
  3. "자동 수정" 버튼을 클릭합니다
  4. 출력 필드의 "복사" 버튼을 클릭합니다
  5. 복사된 내용이 정확히 포맷팅된 JSON인지 확인합니다

테스트 케이스 1: 기본 JSON

입력 (잘못된 형식)

{'name': "iPhone 17", 'price': 1200000, features: ["AI", "5G", 'Camera']}

예상 출력 (포맷팅된 JSON)

{ "name": "iPhone 17", "price": 1200000, "features": [ "AI", "5G", "Camera" ] }

테스트 케이스 2: 중첩된 JSON

입력

{ "cf_id": "CF_000001", "current_stage_name": "scenario_development", "timestamp": "2025-09-01T13:30:32Z", "schema_version": "1.0.0", "transfer_ready_for_next_stage": true, "next_stage_name": "visual_concept", "source_cf_id_for_current_stage": "CF_000001_2-1", "cf_metadata": { "product_name": "iPhone 17", "campaign_purpose": "제품 론칭", "confirmed_genre": "Tech", "duration_minutes": 0.5, "duration_seconds": 30, "target_audience": "20대", "emotional_tone": "영감적", "visual_tone": "미래적", "context_analysis": { "temporal_context": "현대, 가까운 미래", "cultural_context": "AI 기술이 일상에 통합되는 문화", "market_context": "개인화된 AI 경험을 중시하는 스마트폰 시장" } }, "scenario_data": { "source_data_references": { "planning_data_id": "CF_000001_2-1", "conti_structure_type": "opening-development-closing" }, "treatment_data": {} } }

테스트 케이스 3: 한 줄 JSON (압축)

입력

{"id":1,"name":"테스트","data":{"value":100,"status":"active"},"items":[1,2,3]}

예상 출력 (정렬 버튼 클릭 후)

{ "id": 1, "name": "테스트", "data": { "value": 100, "status": "active" }, "items": [ 1, 2, 3 ] }

복사 기능 검증

검증 항목:

  • ✅ 복사 버튼 클릭 시 "복사완료" 메시지 표시
  • ✅ 2초 후 자동으로 "복사" 텍스트로 복귀
  • ✅ 클립보드에 정확한 JSON 포맷 저장
  • ✅ 들여쓰기와 줄바꿈 유지
  • ✅ 한글 문자 깨짐 없음
  • ✅ 특수문자 이스케이프 처리 정상
